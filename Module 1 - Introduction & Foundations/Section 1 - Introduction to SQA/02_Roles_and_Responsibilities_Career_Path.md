## Typical QA Roles in Teams

### 1. QA Engineer / QA Tester (Entry Level)

**Responsibility Level**: Junior / Entry

**Primary Duties**:
- Execute test cases
- Report bugs
- Create basic test cases
- Perform manual testing

**Daily Activities**:
```
Morning:
- Check assigned test cases
- Execute tests
- Document results

Afternoon:
- Report bugs found
- Discuss with team
- Update test documentation
```

**Skills Needed**:
- Basic understanding of SDLC
- Communication skills
- Attention to detail
- Basic SQL knowledge (optional)

**Salary Range**: Please Search according to the current time, because it changes..

**Career Progression**: 
QA Tester → QA Analyst → Senior QA Engineer

**Real Example**:
```
John starts as a QA Tester:
- Tests login feature
- Tests payment functionality
- Reports bugs found
- Documents test results
- Learns from experienced team members
```

## Skills Required to Become a Successful QA Engineer

### Technical Skills

#### 1. **Testing Knowledge**
- Understanding of SDLC and STLC
- Different testing types (functional, non-functional)
- Test case design techniques
- Risk-based testing

**Example**:
```
Test Case Design:
- Boundary value analysis
- Equivalence partitioning
- Error guessing
- State transition testing
```

#### 2. **SQL / Database Knowledge**
- Write basic SQL queries
- Verify data in databases
- Understand relationships

**Example Query**:
```sql
-- Verify user login attempt
SELECT * FROM login_logs 
WHERE user_id = 123 
AND login_date = '2024-01-15'
ORDER BY login_time DESC;
```

#### 3. **Tools Knowledge**
- JIRA (bug tracking)
- Postman (API testing)
- Selenium (automation)
- Git (version control)

#### 4. **Programming Skills** (Intermediate/Senior)
- JavaScript, Python, or Java
- Writing test scripts
- Understanding code logic
- Debugging

**Example - Automated Test**:
```javascript
// Simple test script
const test = async () => {
  // Navigate to login page
  await page.goto('https://example.com/login');
  
  // Fill credentials
  await page.fill('#username', 'user@example.com');
  await page.fill('#password', 'password123');
  
  // Click login
  await page.click('#loginBtn');
  
  // Verify redirect
  expect(page.url()).toContain('/dashboard');
}
```

#### 5. **API Testing Knowledge**
- HTTP methods (GET, POST, PUT, DELETE)
- JSON/XML understanding
- API testing tools (Postman)
- Response validation

---

### Soft Skills

#### 1. **Communication**
- Clearly explain bugs
- Listen to developers
- Ask right questions
- Write clear documentation

**Bad Bug Report**:
```
"Login not working"
```

**Good Bug Report**:
```
"Login fails with valid email credentials"
- Steps: Open app → Click login → Enter 'user@example.com' → Enter 'password123' → Click login
- Expected: Redirect to dashboard
- Actual: Shows error "Invalid credentials"
- Screenshot: [attached]
- Reproducibility: Always
```

#### 2. **Problem-Solving**
- Identify root cause of bugs
- Think of edge cases
- Suggest improvements
- Troubleshoot issues

**Example**:
```
Bug: "Form submission fails randomly"

Investigation:
1. Does it fail on all browsers? No, only on Chrome
2. Does it fail on all devices? No, only on mobile
3. Root cause: Screen size issue
Solution: Fix CSS for mobile screens
```

#### 3. **Attention to Detail**
- Notice small defects
- Precise documentation
- Careful testing
- Verify requirements

#### 4. **Time Management**
- Prioritize tasks
- Meet deadlines
- Organize work
- Plan sprints

#### 5. **Teamwork**
- Collaborate with developers
- Help other testers
- Share knowledge
- Respect feedback

#### 6. **Critical Thinking**
- Think "what if?"
- Question assumptions
- Suggest improvements
- Find creative bugs

## Certifications in QA

### ISTQB (International Software Testing Qualifications Board)

#### 1. **ISTQB Certified Tester Foundation Level**
- **What it covers**: 
  - SDLC fundamentals
  - Testing types
  - Test management
  - Tools and automation basics
  
- **Prerequisites**: None

- **Exam details**:
  - 40  multiple choice questions
  - 60 minutes  (with extra time granted for non-native languages)
  - 26/40 passing score (65% or higher)
  
- **Cost**: $150 - $300

- **Value**: 
  - Entry-level certification
  - Shows foundational knowledge
  - Good for getting first QA job
  - Worldwide recognition

- **Study time**: 20-40 hours

**Real Example**:
```
Sarah got ISTQB Foundation Level:
- Studied for 4 weeks (weekends)
- Passed exam with 82%
- Used certification in resume
- Got promoted to QA Analyst
```

---

#### 2. **ISTQB Certified Tester Advanced Level**
- **What it covers**:
  - Test analysis
  - Test design
  - Test management
  - Advanced techniques
  
- **Prerequisites**: Foundation Level + 2 years experience

- **Specializations**:
  - Test Analyst
  - Test Manager
  - Technical Test Analyst

- **Cost**: $200 - $350

- **Value**: 
  - Mid-level certification
  - Competitive advantage
  - Needed for promotions
  - Global recognition

---

### CSTE (Certified Software Test Engineer)
- Offered by Quality Assurance Institute (QAI)
- Covers practical testing experience
- Requires 4+ years testing experience
- Industry recognition

### Other Certifications
- **ISTQB Mobile Tester**: For mobile app testing
- **ISTQB Agile Tester**: For Agile environments
- **ISTQB Performance Tester**: For performance testing

## How QA Collaborates with Other Teams

### QA & Developers Collaboration

```
Typical Interaction:
1. Developer: "Feature ready for testing"
2. QA: "Looks good, I'll test and report any bugs"
3. Developer: Reviews QA bug report
4. Developer: "Fixed the bug"
5. QA: "Re-testing to confirm fix... Fixed! "
```

**Communication Methods**:
- Daily standups
- Slack messages
- Bug discussions
- Code reviews
- Design reviews

**Example Conversation**:
```
QA: "Found a bug: login fails with special characters"
Developer: "Which special characters? Can you share the exact input?"
QA: "Yes, tried: user@#$%.com"
Developer: "Ah, I see. Our validation regex doesn't handle that. Let me fix."
Developer: "Fixed, ready for re-test"
QA: "Tested with same input, working now! "
```

---

### QA and Product Manager Collaboration

```
Typical Interaction:
1. PM: "Here's the new feature requirement"
2. QA: "Can you clarify the edge cases?"
3. PM: "Good question. If input is empty, show error."
4. QA: "Got it. I'll test both scenarios"
```

**Communication Methods**:
- Requirement reviews
- Clarification calls
- Feedback on testability
- Metrics reporting
- Risk assessment

---

### QA & DevOps Collaboration

```
Typical Interaction:
1. QA: "Can you set up test environment?"
2. DevOps: "Sure, what configuration do you need?"
3. QA: "Production-like setup with test database"
4. DevOps: "Done. Here's the access details"
5. QA: "Can run tests now. Thanks!"
```
